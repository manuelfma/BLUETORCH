# Product Feature Review — Blue Torch.me

## 1. Hypothesis
Investment professionals will trust AI-generated diligence risks when each item includes source citation, confidence level, evidence type, missing information, and reviewer approval state.

## 2. MVP Experiment
Run a concierge test with the landing page, Stitch prototype, and sample risk register. Ask target users to review one or two risk findings and decide whether they would approve, reject, edit, or escalate each item.

## 3. Target User
VC, growth equity, and private equity associates or principals who perform first-pass diligence and prepare materials for partners or investment committee.

## 4. Feedback Questions
- How would you verify this risk before adding it to an IC memo?
- What information is missing?
- Would you approve, reject, edit, or escalate this item?
- Does the citation make the output more trustworthy?
- Would this save time in your current workflow?

## 5. Key Learning
Expected learning:

The team needs to know whether source-linked review states are enough to make AI-generated diligence useful and credible for investment workflows.

To be completed after user testing:

- user quotes
- trust score
- most common objection
- missing feature requests
- decision: iterate, pivot, or continue

## 6. Next Changes
If validated:

- Build one live risk-extraction workflow.
- Prioritize source preview and reviewer states.
- Defer advanced memo export and IC Debate.

If invalidated:

- Reduce AI autonomy.
- Make the product an evidence extraction and memo-organization tool first.
- Re-test with narrower claims.
